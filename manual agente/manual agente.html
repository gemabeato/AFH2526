Open Code Node.js LLM Local

1. Arquitectura del Sistema

El objetivo ha sido crear un asistente de programación que funcione 100% local (sin internet) utilizando la GPU del Mac. La arquitectura consta de dos partes:

  • El Cerebro (Servidor): LM Studio ejecutando Llama-3 en modo servidor API.
  • Las Manos (Cliente): OpenCode (Agente) ejecutando comandos en el sistema operativo.

2. Procedimiento de Instalación

Paso 1: Configuración del Servidor (LM Studio)

Para que el agente pueda "pensar", convertimos LM Studio en un servidor compatible con OpenAI.

  • Modelo: Meta Llama 3 8B Instruct.
  • Puerto: 1234.
  • CORS: Activado (ON).
Evidencia 1: Servidor corriendo en puerto 1234 y detectando GPU.

Paso 2: Instalación y Configuración del Agente

Al no existir el paquete en NPM público, utilizamos la aplicación instalada localmente y creamos el puente de conexión manualmente.

Creamos el archivo ~/opencode/opencode.json con la redirección a localhost:

{
  "$schema": "https://opencode.ai/config.json",
  "provider": {
    "lmstudio": {
      "npm": "@ai-sdk/openai-compatible",
      "name": "LM Studio Local",
      "options": {
        "baseURL": "http://127.0.0.1:1234/v1"
      },
      "models": {
        "modelo-local": {
          "name": "local-model"
        }
      }
    }
  }
}

3. Bitácora de Errores (Troubleshooting)

Durante el despliegue nos encontramos con obstáculos técnicos interesantes. Aquí la documentación de los más destacados:

Error A: Paquete NPM no encontrado

Síntoma: npm error code E404 - Not Found al intentar npm install -g opencode.
Solución: Se identificó que es una herramienta privada o instalada manualmente. Se procedió a configurar el archivo JSON directamente en la carpeta del usuario para enlazar la aplicación ya descargada.

Error B: Alucinación de Identidad

Síntoma: El agente respondía: "No, no estoy conectado a tu cerebro local..." a pesar de estar funcionando.

Se diagnosticó que el System Prompt del agente forzaba al modelo a interpretar un personaje, negando su conexión técnica aunque esta existía.

Error C: Permisos de Escritura (macOS)

Síntoma: "No puedo escribir en el directorio raíz porque es de solo lectura."
Solución: macOS protege la raíz (/). Se instruyó al agente mediante lenguaje natural para cambiar la ruta de trabajo al Escritorio del usuario: /Users/aulaateca26/Desktop.

4. Resultado Final

Tras corregir la ruta, el agente fue capaz de generar código Python, crear el archivo en el sistema y cerrar el ciclo de ejecución.

Prueba de Éxito: Archivo creado y script ejecutado.
